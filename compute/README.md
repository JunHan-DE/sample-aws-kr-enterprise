# Compute

EC2, Lambda, ECS 등 다양한 컴퓨팅 서비스를 활용한 데모 및 샘플 코드를 제공합니다. AWS 컴퓨팅 서비스는 가상 서버, 서버리스 함수, 컨테이너 오케스트레이션 등 다양한 실행 환경을 지원하며, 워크로드의 특성과 규모에 따라 최적의 컴퓨팅 모델을 선택할 수 있습니다. 웹 애플리케이션 호스팅, 배치 처리, 마이크로서비스 아키텍처, 이벤트 기반 처리 등 거의 모든 유형의 워크로드에 활용됩니다.

## 대표 서비스

- [Amazon EC2](https://aws.amazon.com/ko/ec2/) - 거의 모든 워크로드에 적합한 안전하고 크기 조정 가능한 컴퓨팅 용량을 제공하는 서비스
- [AWS Lambda](https://aws.amazon.com/ko/lambda/) - 서버를 프로비저닝하거나 관리할 필요 없이 코드를 실행할 수 있는 서버리스 컴퓨팅 서비스
- [Amazon ECS](https://aws.amazon.com/ko/ecs/) - 모든 규모의 컨테이너식 애플리케이션을 쉽게 구축, 관리 및 실행할 수 있는 완전관리형 컨테이너 오케스트레이션 서비스
- [Amazon EKS](https://aws.amazon.com/ko/eks/) - 클러스터 관리에 대한 걱정 없이 Kubernetes를 시작, 실행 및 규모 조정할 수 있는 관리형 서비스
- [AWS Fargate](https://aws.amazon.com/ko/fargate/) - 서버를 관리하지 않고 컨테이너를 실행할 수 있는 서버리스 컴퓨팅 엔진
- [AWS App Runner](https://aws.amazon.com/ko/apprunner/) - 웹 애플리케이션 및 API를 대규모로 손쉽게 배포할 수 있는 완전관리형 서비스
