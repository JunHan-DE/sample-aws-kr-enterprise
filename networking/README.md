# Networking

VPC, CloudFront, Route 53 등 네트워킹 및 콘텐츠 전송 서비스를 활용한 데모 및 샘플 코드를 제공합니다. AWS 네트워킹 서비스는 클라우드 내 가상 네트워크 구성, 글로벌 콘텐츠 전송, DNS 관리, API 관리, 로드 밸런싱 등 네트워크 인프라의 핵심 요소를 제공합니다. 안전한 네트워크 격리, 멀티 리전 아키텍처 구성, 하이브리드 클라우드 연결, 고가용성 애플리케이션 구축 등 엔터프라이즈 환경에 필수적인 기능을 지원합니다. 글로벌 AWS 백본 네트워크를 활용하여 낮은 지연 시간과 높은 처리량을 보장합니다.

## 대표 서비스

- [Amazon VPC](https://aws.amazon.com/ko/vpc/) - 논리적으로 격리된 가상 네트워크에서 AWS 리소스를 정의하고 시작할 수 있는 서비스
- [Amazon CloudFront](https://aws.amazon.com/ko/cloudfront/) - 짧은 지연 시간과 빠른 전송 속도로 전 세계 사용자에게 안전하게 콘텐츠를 전송하는 CDN 서비스
- [Amazon Route 53](https://aws.amazon.com/ko/route53/) - 최종 사용자를 인터넷 애플리케이션으로 라우팅하는 안정적이고 비용 효율적인 DNS 서비스
- [Amazon API Gateway](https://aws.amazon.com/ko/api-gateway/) - 모든 규모의 API를 생성, 유지 관리 및 보호할 수 있는 완전관리형 서비스
- [Elastic Load Balancing](https://aws.amazon.com/ko/elasticloadbalancing/) - 네트워크 트래픽을 분산하여 애플리케이션의 확장성과 가용성을 개선하는 서비스
- [AWS Transit Gateway](https://aws.amazon.com/ko/transit-gateway/) - Amazon VPC, AWS 계정 및 온프레미스 네트워크를 단일 게이트웨이에 연결하는 서비스
